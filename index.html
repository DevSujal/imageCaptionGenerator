Image Caption Generator

Developer usage: how to call the caption API, request shape, examples, and response format.

POST /api/v1/caption
Content-Type: multipart/form-data
What it does

Upload an image file and receive a concise one-line caption describing the image's main subject and notable attributes. The server sends the image to Google GenAI and returns the generated caption.

Request structure
  • Method: POST
  • Path: /api/v1/caption
  • Content-Type: multipart/form-data
  • Form field: image — the binary file (jpeg, png, gif, webp supported)
Examples
curl (Linux/macOS/Windows w/ curl): 
curl -X POST 'http://localhost:3000/api/v1/caption' \ -F "image=@/path/to/photo.jpg"
PowerShell (Invoke-RestMethod): 
Invoke-RestMethod -Uri 'http://localhost:3000/api/v1/caption' -Method Post -Form @{ image = Get-Item 'C:\path\to\photo.jpg' }
Browser (fetch): 
// JavaScript (client)
const form = new FormData();
form.append('image', fileInput.files[0]);
const res = await fetch('/api/v1/caption', { method: 'POST', body: form });
const json = await res.json();
console.log(json.caption);
Response
{
  "caption": "A concise one-line caption describing the image"
}
Errors
  • 400 — when the `image` field is missing: { "error": "No image file uploaded" }
  • 500 — server or GenAI failure: { "error": "Failed to generate caption" }
Environment (server)

The server requires credentials to call the GenAI models. Provide them on the server (do not expose in client code): 

GEMINI_API_KEY="<your_key_here>"
ORIGIN="*"
Do not commit secrets. Use environment variables or cloud secret managers. The client should never see these keys.
Client tips
  • Resize large images on the client if possible to reduce upload time and costs.
  • Show progress and friendly messages while the caption is generated.
  • Implement retries with exponential backoff for transient server errors.
  • Protect the endpoint with authentication and rate limits in production.
